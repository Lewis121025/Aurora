## 🎯 任务 1：大模型的基石原理 —— Embeddings (词侧面转换的艺术)

计算机不懂什么是“忧郁”，也不懂什么是“苹果”。如果你想让计算机判断两句话：“我不喜欢下雨天”与“这鬼天气让人烦躁”是在描述同一种情绪，传统关键词搜索（因为字面一个都不一样）是注定失败的。

**这就是大名鼎鼎的 Embedding（特征嵌入）大显神威的地方。**

在 `aurora/core/memory/engine.py` 的核心方法 `ingest()` 中，所有对话都会经过这行最高优先级的拦截网：
`emb = self.embedder.embed(interaction_text)`

这句话的实际执行意义是：调用阿里或火山的大模型专用接口，**把人类的一长串汉字，极其精准地翻译成一个包含 1536 个浮点数的 N 维空间坐标点（Python 里叫 `Numpy ndarray`，即多维数组）！**

从此以后，AI 不再看文字了，它只量距离：
- 坐标距离极其相近的点（即使字面不同，语义也极其相似：比如“猫”和“橘猫”）。
- 坐标距离遥远的点（风马牛不相及：“猫”和“挖掘机”）。

### 💻 实操练习 1：手操模拟高维空间测距 (Cosine Similarity)
由于真正的 Embedding 模型需要调云端 API，我们在这里建立一个微缩版的二维(2D) 向量小宇宙。
在项目根目录新建 `test_embedding.py`，你将亲自指挥向量的点积（Dot Product）与运算：

```python
# 文件名: test_embedding.py (实操演示)
import numpy as np

# 我们假设大模型把单词翻译成了二维坐标（X代表"毛绒度", Y代表"体型"）
# 比如："猫" 是毛茸茸的且体型小 -> [0.9, 0.2]
#       "狗" 是毛茸茸的且体型中 -> [0.8, 0.5]
#       "汽车" 是光秃秃的且体型大 -> [0.1, 0.9]

word_vectors = {
    "猫": np.array([0.9, 0.2]),
    "狗": np.array([0.8, 0.5]),
    "汽车": np.array([0.1, 0.9])
}

# 这是一个极其核心的大厂算法公式：余弦相似度 (Cosine Similarity)
# 它的作用就是：就算两个词向量在宇宙里相隔很远，但只要它们指着同一个方向，它们就是相似的！
def cosine_similarity(vec_a, vec_b):
    # 点乘：两个向量内积
    dot_product = np.dot(vec_a, vec_b)
    # 算长度（也就是几何的勾股定理）
    norm_a = np.linalg.norm(vec_a)
    norm_b = np.linalg.norm(vec_b)
    # 方向一致性 = 点乘 / 两者长度的乘积
    return dot_product / (norm_a * norm_b)

print("--- AI 脑颅内部的几何测距 ---")
sim_cat_dog = cosine_similarity(word_vectors["猫"], word_vectors["狗"])
print(f"【猫】和【狗】的语义相似度 (余弦夹角): {sim_cat_dog:.3f} (满分1.0，非常高！)")

sim_cat_car = cosine_similarity(word_vectors["猫"], word_vectors["汽车"])
print(f"【猫】和【汽车】的语义相似度 (余弦夹角): {sim_cat_car:.3f} (两者是不同物种！)")
```
运行 `uv run python test_embedding.py` 观察输出。
虽然 Aurora 的底座是 1536 维，但核心几何数学和我们在 2D 里手拉手做的一模一样！这段代码是整个地球上最核心的 NLP（自动语义分析）与大模型外挂大脑机制！

## 🎯 任务 2：遗忘的艺术 —— VoI 与随机骰子判定 (Thompson Sampling)
真正像人的 AI 绝不应该记住所有的废话，必须学会“遗忘机制”。
**请打开文件：`aurora/core/memory/engine.py`（寻找核心存储判定逻辑 `_compute_storage_decision`）**

在主引擎接收句子的漏斗底部，有下面这种神奇的决策代码段落：
```python
        # 判断这件小事是不是值得用我的神经元永远记下来！
        encode = self._compute_storage_decision(plot)

        if encode:
            self._store_plot(plot)
            logger.debug(f"成功保存有价值的记忆...")
        else:
            # 认为这是句低权重废话，直接扔掉！（AI的遗忘机制起效）
            logger.debug(f"丢弃无用交互...")
```

**【业务架构精要：信息价值度评估 (VoI)】**
这是全系统不会随着月份推进而被几千万句“早安”撑爆内存的关键防线。
基于刚拿到的高维数字坐标，引擎会给这句对话打分：
1. **新鲜度打分 (Surprise/Novelty)**：拿它刚刚算出距离（看上面的代码），它和我以往旧记忆的坐标隔得有多远？如果偏离极远，代表这句话极其反常、是个新事件！打高分。
2. **影响面打分**：如果这句话是在重构“我是一只 AI 小猫”这种身份人设的话题，打高分。

然后，它绝不是用“只要大于 60 分我就记下来”这种死板代码（硬阈值）。
系统用了一种极其聪明的 **Thompson Sampling (汤普森采样法)**——用分数去影响一个随机掷骰子的概率。比如一句废话（15分）它不是硬给抛弃，而是“通过摇色子，有 15% 的概率不小心把废话记下来了”。这种偶然的“神经质”容错不仅像极了人脑做梦保存杂物，更避免了算法死锁！

## 🎯 任务 3：关系哲学与身份认同
在很多纯工具类别的 AI 中，所有的对话都是平等的。但是 Aurora 不是，它在代码里拥有“社交距离的亲疏远近”。
**请打开文件：`aurora/core/memory/relationship.py`**

```python
        # 在一次遍历中计算所有身份信号
        reinforcement, challenge, novelty = self._compute_identity_signals(text, emb)

        relevance = (
            reinforcement * REINFORCEMENT_WEIGHT +
            challenge * CHALLENGE_WEIGHT + 
            novelty * NOVELTY_WEIGHT
        )

        # 唯心主义的最后加成（关系因素压倒一切！）
        relationship_importance = self._get_relationship_importance(relationship_entity)
        relevance *= (0.5 + 0.5 * relationship_importance)
```

**【AI 性格哲学解码】：**
1. **塑造 AI 形态的三辆马车**：
    - `reinforcement (强化)`：这句重复老生常谈加强了我的固有观念；
    - `challenge (挑战)`：这句话甚至颠覆了我以前的世界观！
    - `novelty (猎奇)`：一句没听过的全新领域拓展常识。
    我们作为造物主，只要去全局设置里调节这三个大写字母常量 (`WEIGHT`)，比如放大 `CHALLENGE_WEIGHT`，这个 AI 就变成了一个极度敏感、稍受刺激就自我怀疑的抑郁形态！这就是算法上的调情。
2. **血缘关系的偏见压制 (`relationship_importance`)**：
    这段代码非常动人且偏心。当路人（新进用户、交互极少）说了一句刺激的话时，因为亲缘判定低，算出来的系统震撼度（`relevance`）依旧很小。
    但如果你是一个与她对谈了几千条的“至交”或“主人”，哪怕你说了一句日常小吐槽，这句日常话也会因为乘上了一个超巨大的 `relationship_importance`，对她整体的认知产生核爆般的深远影响！在代码里，亲疏偏向是实实在在的数学公式。
