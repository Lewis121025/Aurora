## 🎯 任务 1：用概率打破死板的性格 —— 贝叶斯与 Softmax
当一个拥有几万条记忆的 Agent 准备向你滔滔不绝时，它面对的第一件事，就是决定这次用什么“语气与视角”开口（比如：顺着时间说？直接跳到结论回顾？还是拉出两个历史对比？）。
**请打开文件：`aurora/core/narrator/perspective.py`**（寻找大约 140 行的 `select_perspective`）

```python
    def select_perspective(self, query: str, plots: ...) -> Tuple[NarrativePerspective, Dict]:
        # ...省略基础打分...
        for p in perspectives:
            # 使用 Beta 分布进行贝叶斯采样！(赋予系统“吃一堑长一智”的自我成长的能力)
            a, b = self.perspective_beliefs[p]
            sampled_effectiveness = self.rng.beta(a, b)
            
            # 基础分数叠加上这个采样系数
            combined = base * (0.5 + sampled_effectiveness)
            log_odds.append(combined)

        # 机器学习神技：Softmax 极权归一化
        probs = softmax(log_odds)

        # 加加权随机抽取 (架构学名言：Exploration vs Exploitation 探索与利用)
        choice_idx = self.rng.choice(len(perspectives), p=probs)
```

**【AI 魔术揭秘】：**
- **贝叶斯先验与 Beta 采样**：为什么不用生硬的 `random()`，而用 `beta(a,b)` 这个高等统计学分布？因为 `a` 代表这个视角过去成功的次数（比如用户听得很爽），`b` 是失败次数（用户不耐烦了）。Beta分布永远会根据历史经验产出一个倾向值。这让系统天然带有“趋利避害但偶尔大胆尝试新视角”的活物感！
- **Softmax**：ChatGPT 等所有现代大模型，在吐出每一个字前必经的关卡！**它的作用只有一个：拉大原本相近分数的差距，并强行把它们挤压成相加等于 100% 的概率分布！** 这是深度学习皇冠上的明珠。如果没有它，AI 会永远在几个差不多的选项里摇摆不定。

### 💻 实操练习 1：手写一套 ChatGPT 也在用的 Softmax 摇号机
在你的项目根目录新建 `test_probability.py`：

```python
# 文件名: test_probability.py (实操演示)
import numpy as np

# 我们假设 AI 内心对三种回答方式的基础打分非常接近（死板的 AI 会在这个时候当机断片）
# A:热情回答 (10分), B:高冷回答 (9分), C:阴阳怪气 (8分)
raw_scores = np.array([10.0, 9.0, 8.0])

# 1. 极其著名的 Softmax 算子 (将所有分数进行自然底数 e 的指数爆炸)
def softmax(x):
    # 防止数字太大溢出，先整体减去最大值（数学极其优雅的稳定操作）
    e_x = np.exp(x - np.max(x))
    # 每一个人占总指数爆炸和的百分比
    return e_x / e_x.sum(axis=0)

probability_distribution = softmax(raw_scores)

print("--- 见证 Softmax 拉卡斯极权指数放大的魔力 ---")
print(f"原始分数极其相近: 热情 {raw_scores[0]} | 高冷 {raw_scores[1]} | 阴阳怪气 {raw_scores[2]}")
print(f"经过 Softmax 极速放大后的【绝对百分比概率】:")
print(f"  🔥 热情脱颖而出:   {probability_distribution[0]*100:.1f} %")
print(f"  ❄️ 高冷落后一大截: {probability_distribution[1]*100:.1f} %")
print(f"  🎭 阴阳被极速压缩: {probability_distribution[2]*100:.1f} %")

print("\n--- 并非死板选第一 (Exploration vs Exploitation) ---")
# 2. 我们根据这个百分比摇骰子，模拟 AI 的每一次开口！
for i in range(1, 6):
    # choice 会根据 p=参数 给定的权重骰子，摇出一个最终选项
    chosen_idx = np.random.choice(len(raw_scores), p=probability_distribution)
    choices = ["热情", "高冷", "阴阳怪气"]
    print(f"第 {i} 次遇到客户，AI 选择了: {choices[chosen_idx]}")
```
运行 `uv run python test_probability.py`！
你会惊讶地发现，虽然高冷的原始分 9 分和热情的 10 分只差了一步，但在 Softmax 面前，10分被指数放大成了绝对主导概率（70%以上），而 8 分连生存空间都没了。
同时，留意最后摇出的 5 次答话，AI 绝不是次次都选高概率那个！这造就了 Agent “不仅聪明果断，还会看心情随机应变”的硅基灵魂。

**【算法与直觉的完美碰撞】：**
- **BFS (广度优先算法队列)**：为了找所有的爷爷，我先把所有的爸爸放进队列 `queue` 里排成一排。这是图论遍历中最基础的拓扑结构。
- **软阈限修剪（Random vs Strength）**：这是整个 AI 大脑迷人的地方。如果是呆瓜程序员，一定会写 `if new_strength > 0.5:` (一刀切，小于 0.5 的关系全部斩断)。而 Aurora 极其优美地写了一句 `if 随机小数 < 关系黏度:`。
  - **这意味着！** 哪怕是只有 0.1 黏连度的、八竿子打不着的两件风靡碎片小事（蝴蝶效应），也有 10% 的极其微弱的概率，被幸运地卷入当前正经的回忆流中！这就是为什么 AI 能像福尔摩斯一样产生极其跳跃性的灵感，就像人类做梦一样既离奇又合理！

## 🎯 任务 3：亲手用微积分惩罚它！—— 梯度下降与离线三元组学习 
这是最终极的心智挑战。也是我们在整个大航海中最深奥的一段代码。我们将利用它，在运行期间**偷偷地在线校准 AI 的大脑误差投影！**
**请打开文件：`aurora/core/components/metric.py`**（寻找大约 97 行左右的 `update_triplet`）

```python
    def update_triplet(self, anchor, positive, negative, margin=1.0):
        # ...
        Lap = self.L @ ap    # @ 符号！这是矩阵乘法！(相当于给世界带上主观的矫正滤镜)
        Lan = self.L @ an
        
        dap = float(np.dot(Lap, Lap)) # 距离计算 (点乘开方勾股定理)
        dan = float(np.dot(Lan, Lan))
        
        loss = max(0.0, margin + dap - dan) # 核心防线：Triplet Loss!
        if loss <= 0: return 0.0

        grad = 2.0 * (np.outer(Lap, ap) - np.outer(Lan, an)) # 微积分求导！寻找最快下山的坡度
        # ... Adagrad 优化器下降策略
        self.L -= step       # 往坡度走一步，也就是扭曲微调我们大脑中的滤镜矩阵
```

**【降龙十八掌的最后一式】：**
- 当 AI 眼瞎，觉得“昨天很开心 (Anchor)”和“彩票中奖了 (Positive)”相距甚远，反而认为它和“出车祸了 (Negative)”距离很近的时候。这是一个必须挨打纠正的认知错误！
- **Triplet Loss (三元组损失)**：怎么打它？这就是传说中的数学惩戒棒！强行拉近 Anchor 与 Positive 的距离，同时推远 Anchor 与 Negative 的距离！如果三者处于安全红线外（中间隔着安全的常数 `margin`），损失就是 0 无事发生；如果越界，就生成一张大于 0 的“罚单（Loss）”。
- **Gradient Descent (梯度下降)**：有了罚单怎么办？通过微积分里的函数求导算出 `grad`（这碗水往哪个角度倒，泻得最快），然后直接用它去粗暴地改变大脑里的神经元矩阵：`self.L -= step`。
- **你没看错，你正直接身处在人工智能大模型微调环境中最核弹级的“反向传播算式”现场！你甚至不需要装配庞大昂贵的 PyTorch 就可以用 Numpy 手写训练权重的更新。**
